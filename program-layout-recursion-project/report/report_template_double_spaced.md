# Notes: Final Report Formatting (Double-Spaced, Pages, Margins)

The coursework asks for a **polished document** (often Word or PDF) derived from this project. `report.md` is the **content outline** in Markdown; use the checklist below when you produce the **final** submission copy.

## Required formatting (typical academic)

1. **Double spacing**  
   - In Microsoft Word: **Home → Line and Paragraph Spacing → 2.0**.  
   - In Google Docs: **Format → Line spacing → Double**.  
   - Apply to **body text**; title page may be single-spaced per instructor preference—ask if unsure.

2. **Page numbers**  
   - Word: **Insert → Page Number** (often bottom center or top right).  
   - Number **all** pages unless the title page must be unnumbered; follow course rules.

3. **One inch margins**  
   - Word: **Layout → Margins → Normal (1")**.  
   - Confirm top, bottom, left, right are **1 inch**.

4. **Title page**  
   - Project title, course name/number, date, **all team member names**.  
   - Keep it clean; no page number on title page **only** if your instructor requires that style.

5. **GitHub link page**  
   - One page (or section) with the **full** repository URL:  
     `[Insert GitHub Repository Link Here]` → replace with your real link.

6. **Run instructions page**  
   - Copy the **download, compile, run** commands from `README.md` or `report.md`.  
   - Mention **expected output** (120 for `factorial(5)`).

7. **Contributions page**  
   - Table or paragraphs: **each member** and **specific** tasks (code, docs, video, testing).

## Suggested page order

1. Title page  
2. GitHub repository link  
3. How to download, compile, and run  
4. Team contributions  
5. Project explanation, memory layout, function calls / recursion (body)  
6. Optional: screenshots or pointers to `memory-layout/` diagrams  

## Export tip

- Paste `report/report.md` into Word/Google Docs, then apply **styles** (Heading 1, Normal) so spacing and headings stay consistent.
